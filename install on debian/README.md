# OpenWRT on VPS — Install Script for Debian

Скрипт для установки OpenWRT поверх Debian на виртуальный сервер (VPS) с архитектурой x86_64.  
Полностью заменяет ОС хоста: записывает образ OpenWRT на системный диск через initramfs, минуя работающую ОС.

---

## Требования

| Параметр | Значение |
|---|---|
| Хостовая ОС | Debian 11 / 12 / 13 (x86_64) |
| Права | root |
| Архитектура | x86_64 |
| Тип виртуализации | KVM (OpenVZ / LXC не поддерживаются) |
| RAM | от 256 МБ |
| Диск | от 512 МБ |
| Сеть | Статический IP на VPS (рекомендуется) |

> ⚠️ **ВНИМАНИЕ**: Скрипт **необратимо** перезаписывает весь системный диск. Все данные на VPS будут уничтожены. Используйте только на выделенных серверах или свежих VPS без ценных данных.

---

## Использование

> ⚠️ **Важно**: используйте только `curl` для скачивания скрипта. `wget` может отдавать устаревшую версию из-за кеширования GitHub CDN.

```bash
curl -sSL -H 'Cache-Control: no-cache' \
  https://raw.githubusercontent.com/Van-Hoffen/OpenWRT/main/install%20on%20debian/install_openwrt_debian.sh \
  -o install.sh && bash install.sh
```

После запуска скрипт задаст серию вопросов в интерактивном режиме.

## Интерактивная настройка

Скрипт не требует переменных окружения — все параметры запрашиваются в диалоге. Если просто нажать Enter — применится значение по умолчанию.

### Базовые параметры

| Вопрос | По умолчанию |
|---|---|
| SSH port | случайный (10000–59999) |
| LuCI web UI port | случайный (10000–59999, не совпадает с SSH) |
| Root password | случайный 16-символьный |

### WireGuard сервер (опционально)

```
Установить WireGuard сервер? [y/N]:
WireGuard listen port [51820]:
WireGuard internal network [10.8.0.0/24]:
```

Если выбрано `y`:
- Устанавливаются `wireguard-tools`, `luci-proto-wireguard`, `kmod-wireguard`
- Генерируется пара ключей сервера, приватный ключ хранится в `/etc/wireguard/server.key`
- Поднимается интерфейс `wg0` с заданным IP и портом
- Открывается UDP-порт в firewall
- **Публичный ключ сервера** записывается в `/etc/banner` — виден при каждом входе по SSH

### PODKOP + sing-box (опционально)

```
Установить PODKOP (bypass censorship)? [y/N]:
VLESS URL (или Enter для пропуска):
Добавить список 'meta'? [y/N]:
Добавить список 'telegram'? [y/N]:
Добавить список 'twitter'? [y/N]:
Добавить список 'discord'? [y/N]:
Добавить список 'googleai'? [y/N]:
Добавить список 'googleplay'? [y/N]:
DNS для PODKOP — doh или plain [doh]:
```

- Список `russiainside` добавляется **всегда** автоматически
- VLESS URL можно не вводить сейчас — добавить позже в LuCI → **Services → Podkop**
- VLESS URL читается через `read -r`, поддерживает любые спецсимволы

## Что делает скрипт

1. **Задаёт вопросы** — SSH-порт, LuCI-порт, пароль, WireGuard, PODKOP, списки
2. **Определяет сеть** — IP-адрес, маску, шлюз на активном интерфейсе
3. **Устанавливает зависимости** — `curl`, `gzip`, `fdisk`, `openssl` и др.
4. **Авто-определяет последнюю стабильную версию OpenWRT** — через `sha256sums` на сервере OpenWRT
5. **Скачивает образ** `openwrt-*-x86-64-generic-ext4-combined.img.gz` и проверяет SHA256
6. **Монтирует rootfs** и патчит его через `uci-defaults`:

| Файл | Порядок | Назначение |
|---|---|---|
| `10_dns_fix` | первый | DNS 1.1.1.1/8.8.8.8, отключение dnsmasq |
| `20_vps_base` | второй | SSH, LuCI, правила firewall |
| `30_wireguard` | третий | WireGuard wg0 (если выбрано) |
| `40_podkop` | четвёртый | PODKOP + singbox-tmpdir (если выбрано) |

7. **Собирает initramfs** с образом внутри
8. **Перезагружает сервер** — initramfs записывает образ на диск через `dd` и перезагружается в OpenWRT

## После установки

Подключение по SSH:

```bash
ssh root@<WAN_IP> -p <SSH_PORT>
```

Открыть веб-интерфейс LuCI:

```
http://<WAN_IP>:<LUCI_PORT>
```

### Публичный ключ WireGuard

После первого входа публичный ключ сервера отображается прямо в баннере:

```
=== WireGuard Server Public Key ===
aBcDeFgHiJkL...=
====================================
```

Используйте его для настройки клиентов (телефон, ноутбук и т.д.).

### Если VLESS URL не был введён при установке

Добавить позже: LuCI → **Services → Podkop** → вкладка **Main** → поле **Proxy String**.

## Технические детали

### DNS — настраивается автоматически

Начиная с текущей версии скрипт **автоматически** прописывает DNS `1.1.1.1` и `8.8.8.8` через UCI и отключает `dnsmasq` ещё на этапе патча образа (`uci-defaults/10_dns_fix`). Ручная настройка после установки **не требуется** — `apk` работает сразу.

### Совместимость паролей

Скрипт использует `openssl passwd -6` (SHA-512) — формат совместим с busybox/shadow в OpenWRT 21.02+.

### Автоматическое определение версии и проверка образа

Скрипт скачивает `sha256sums` с сервера OpenWRT, извлекает имя файла и контрольную сумму автоматически. Если авто-детект не удался — используется fallback на последнюю известную версию. Также предусмотрен fallback на `-efi` вариант образа если стандартный недоступен.

### Механизм takeover

Образ загружается в initramfs (RAM), после чего до монтирования корневой ФС выполняется:

```bash
gzip -dcq /owrt.img.gz | dd of=/dev/sda bs=4M conv=fsync
```

Это позволяет перезаписать диск, с которого загрузилась система — без Live USB и KVM-консоли.

### /tmp/sing-box — tmpfs

PODKOP хранит кэш в `/tmp/sing-box/cache.db`. Так как `/tmp` — это tmpfs, директория исчезает при перезагрузке. Скрипт создаёт `init.d/singbox-tmpdir` (START=19), который пересоздаёт директорию при каждом старте системы.

## Известные ограничения

- Работает только с **одним сетевым интерфейсом** (WAN = `eth0`)
- После установки **LAN-интерфейс** (`br-lan`) настроен на VLAN `eth0.10` — на большинстве VPS не используется
- **HTTPS для LuCI** не настраивается автоматически — только HTTP
- Протестировано на провайдерах с **virtio-net** (KVM); на XEN могут потребоваться правки сетевого интерфейса
- **Скачивайте скрипт только через `curl`** — `wget` может отдавать устаревшую версию из-за кеширования GitHub CDN
